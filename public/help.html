Help for The Hoick Habit Server

At this point the server can only handle saving JSON data for PouchDB on /habits/:username, where :username is an object keygen with an array of habits.

The server accepts JSON via HTTP POST. For example:

{ username: [ {name: 'Meditate', days: [], reminder: null}, {name: 'Clean', days: [], reminder: '11:30'} ] }

Also, HTTP GET /habits/:username will return JSON for the user and their habits.
